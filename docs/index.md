# Academic Technology: Generative AI API Access Guide

*(For Harvard FAS, GSAS, and GSE affiliates – faculty & students)*

Welcome to your secure, short-term gateway to **OpenAI's powerful Large Language Model (LLM) APIs**, managed by Harvard Academic Technology. This service supports academic research, course work, and exploration—typically for **one academic term or less**.

## What you get

| Service Component | Detail |
| :--- | :--- |
| **API Key:** | Direct access to the **OpenAI API** and models (GPT-3.5, GPT-4, etc) |
| **Starter Notebook:** | No code jupyter notebook guide to getting started using API in python |

---

## ⚠️ Critical Usage & Policy Requirements

> **Please review these fundamental rules before requesting an API key:**

| Policy Area | Details |
| :--- | :--- |
| **Eligibility:** | Only FAS, GSAS, and GSE affiliates are eligible. Affiliates of other schools (HBS, HMS, etc.) should contact their local IT team. |
| **Authentication:** | Request form requires **HarvardKey** login. |
| **API Access:** | API key gives access to only the **OpenAI Direct API** available through Harvard API Portal |
| **Duration:** | Short-term (up to **one academic term per project**) |
| **Usage Limit:** | **$25 of API credit per month** per key, for the duration of the term. |
| **Key Security:** | Do **not** share your API key in public or team spaces (Slack, GitHub, SharePoint, etc.). Each user must have a unique key. |
| **Distribution:** | If given multiple keys (for a class/workshop), **securely distribute** and instruct recipients in safe handling. |
